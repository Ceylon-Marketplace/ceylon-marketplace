# Vercel Blob Storage Setup

This project uses **Vercel Blob Storage** for secure image uploads in listings.

## Configuration

### Prerequisites
- Vercel project connected to this repository
- Vercel Blob Storage created in the Vercel dashboard

### Environment Variables

Vercel Blob automatically manages credentials when your project is connected to Vercel. The necessary environment variable is:

```env
BLOB_READ_WRITE_TOKEN=<auto-generated by Vercel>
```

This token is automatically available at runtime in Vercel deployments. No manual configuration is required if you've already:
1. ✅ Connected the project to Vercel
2. ✅ Created Blob Storage in the Vercel dashboard

## Features

### Image Upload
- **Endpoint**: `POST /api/listings/upload-image`
- **Max file size**: 10MB per image
- **Supported formats**: JPEG, PNG, WebP, GIF
- **Validation**: File type and size checked server-side
- **Image limit**: Max 10 images per listing (validated on both client and server)

### Image Deletion
- **Endpoint**: `DELETE /api/listings/delete-image`
- **Authentication**: Required (JWT token)
- **Authorization**: Only listing owner can delete images
- **Cleanup**: Images removed from both Vercel Blob and database

### Image Management in Listings
- **Create listing**: Upload up to 10 images during listing creation
- **Edit listing**: Add new images or remove existing ones (max 10 total)
- **Cover photo**: First image in list is displayed as cover photo
- **Image ordering**: Images can be reordered via drag-and-drop in the UI

## Storage Structure

Images are organized by user and upload timestamp:
```
listings/{userId}/{timestamp}-{randomString}.{ext}
```

All images are stored with public access for serving to buyers.

## Implementation Details

### Components
- **ImageUploader** (`src/components/image-uploader.tsx`): React component with drag-and-drop, progress tracking, and validation
- **Create Listing** (`src/app/(main)/listings/create/page.tsx`): Initial image upload
- **Edit Listing** (`src/app/(main)/listings/[id]/edit/page.tsx`): Image management for existing listings

### API Routes
- **Upload** (`src/app/api/listings/upload-image/route.ts`): Handles image upload to Vercel Blob
- **Delete** (`src/app/api/listings/delete-image/route.ts`): Removes images from Blob and database
- **Listing CRUD** (`src/app/api/listings/route.ts`, `src/app/api/listings/[id]/route.ts`): Image validation on listing create/update

### Database
- **ListingMedia** model stores image metadata and URLs
- Images are tied to listings via `listingId` foreign key
- Automatic cleanup of media records when listing is deleted

## Local Development

For local development, you'll need the `BLOB_READ_WRITE_TOKEN`. To get it:

1. In Vercel dashboard, go to your project settings
2. Navigate to "Storage" → "Blob"
3. Copy the token from the "Tokens" section
4. Add to `.env.local`:
   ```env
   BLOB_READ_WRITE_TOKEN=your_token_here
   ```

1. **Upload**: Create a listing and upload images via the UI
2. **Verify**: Images display in the listing preview with proper ordering
3. **Delete**: Remove images from edit page and verify they're gone from Blob
4. **Limit**: Attempt to upload 11 images and verify rejection
5. **Formats**: Test different image formats (JPEG, PNG, WebP, GIF)
6. **Size**: Test 10MB+ file and verify rejection

## Troubleshooting

### Images not uploading
- Check browser console for CORS errors
- Verify `BLOB_READ_WRITE_TOKEN` is set in production
- Ensure Vercel Blob is enabled in project settings

### Upload progress stuck
- Check network tab in DevTools
- Verify file size is under 10MB
- Confirm image format is supported

### Delete fails
- Verify you own the listing
- Check server logs for blob deletion errors
- Database record deletion should still occur even if blob deletion fails

## Security Considerations

- All images are scanned server-side for type and size
- File type validated from MIME type, not extension
- Unique filenames prevent collisions
- Only listing owners can delete images
- Public read access allows buyers to view images
- JWT authentication required for upload/delete operations
